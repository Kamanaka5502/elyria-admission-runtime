# Implementation Pilot

## Purpose

The pilot converts scan findings into an enforceable proof corridor for one bounded consequence-bearing path.

## Good Pilot Candidates

- payment release
- production deployment
- AI recommendation handoff
- customer escalation
- vendor approval
- policy exception
- data access request
- model update approval

## Pilot Outputs

- consequence-node model
- authority rules
- evidence requirements
- refusal matrix
- revalidation triggers
- deterministic verdict engine
- receipt structure
- replay test
- executive pilot report

## Acceptance Criteria

The pilot is acceptable only if it proves:

1. inadmissible movement cannot silently bind
2. missing evidence produces HOLD or REFUSE, not implicit approval
3. refusal reasons are explicit
4. authority scope is checked before consequence movement
5. receipts are generated for verdicts
6. replay can reproduce verdict basis
7. black paths are detected or eliminated

## Boundary

The pilot is not a full enterprise rollout. It proves the first governed corridor.
